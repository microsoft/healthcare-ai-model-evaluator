App Structure & Entry:
index.tsx -> App.tsx -> [AuthProvider -> ReduxProvider -> Router -> Navigation]
  ├─ Login/Login.tsx (uses: Login.scss, authConfig.ts, AuthContext)
  ├─ Arena/* (uses: Arena.scss, arenaReducer.ts, trialService.ts)
  ├─ Rankings/Rankings.tsx (uses: rankingService.ts, LoadingOverlay)
  └─ Admin/* (uses: ManagementLayout.tsx + individual management components)

State Management Hierarchy:
store/store.ts (Root Redux Store)
  ├─ reducers/
  │   ├─ userReducer.ts (auth user management)
  │   ├─ dataReducer.ts (datasets)
  │   ├─ modelReducer.ts (AI models)
  │   ├─ clinicalTaskReducer.ts (clinical tasks)
  │   ├─ testScenarioReducer.ts (test scenarios)
  │   ├─ experimentReducer.ts (experiments)
  │   └─ arenaReducer.ts (arena state)
  └─ hooks/
      ├─ useModels.ts
      ├─ useClinicalTasks.ts
      ├─ useExperiments.ts
      ├─ useTestScenarios.ts
      └─ useMongoUserId.ts

Authentication Chain:
config/authConfig.ts (MSAL config)
  ├─ contexts/AuthContext.tsx (auth state & methods)
  ├─ hooks/useAuth.ts (auth hook)
  └─ services/axiosConfig.ts (auth interceptor)
